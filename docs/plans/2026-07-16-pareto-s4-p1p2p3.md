## Cơ chế chung: run-log.jsonl là sổ memo duy nhất

Cả 3 cơ chế đọc/ghi memo qua `_acceptance/<slug>/run-log.jsonl` (sổ máy có sẵn) — không thêm file state mới. Dòng mới (JS thuần trong workflow tính, scribe/main-loop chép):

```jsonl
{"ts","round","evalId","run_id":"<gốc>","exit_code":0,"cmd","carried_from_round":N}          // P1 eval carried
{"ts","round","evalId","kind":"panel","proposal","votes":[{"lens","verdict"}],"inputs_hash","carried_from_round"?}  // P3
{"ts","round","kind":"baseline","evals_hash","non_discriminating":[{"cmd","evals"}],"carried_from_round"?}          // P2
```

An toàn: `evidence-core.loadRunLogIds` bỏ qua dòng không có `run_id` string → dòng panel/baseline vô hại; run_id carried đã nằm trong set từ round gốc → hook L2 PROVENANCE + recheck CI pass tự nhiên; pre-merge chỉ cần report re-pin `verified_commit` mới — không đổi.

## P1 — Delta-scoped staleness re-verify

- `evals.yaml`: eval máy/ui thêm field TÙY CHỌN `paths: [<glob tương đối repo>]`. Thiếu `paths` → eval LUÔN re-run (mặc định bảo thủ, workspace cũ không đổi hành vi).
- Kích hoạt CHỈ khi: staleness guard fire ∧ report trước verdict PASS-family ∧ có `verified_commit` (round fix sau REJECT KHÔNG có anchor → full re-run như cũ).
- Main loop (SKILL): `deltaFiles = git diff --name-only <verified_commit>` (loại `_acceptance/**`). Eval carried khi: có `paths` ∧ delta ∩ paths = ∅ ∧ dòng run-log round trước của nó exit 0.
- Suite commands LUÔN re-run (lưới hồi quy diện rộng + tín hiệu fresh để re-pin).
- Guard chống rỗng: mọi eval carried ∧ suite rỗng ∧ không judgment fresh → BLOCKED "không còn gì fresh để verify — kiểm tra paths/suite_keys".
- Script: args mới `carriedEvals[]` → không spawn agent; block report ghi run_id gốc + `verified_at` gốc + dòng `carried_from_round: N` (minh bạch, không giả); run-log ghi dòng carried cho round hiện tại.

## P2 — Baseline A/B chạy 1 lần (per trạng thái evals)

- Main loop tính `evalsHash = sha256(evals.yaml)`; đọc dòng `kind:"baseline"` cuối trong run-log: hash trùng → `runBaseline:false` + `carriedAnalyst` từ dòng đó; khác/chưa có → `true`.
- Script: `runBaseline:false` → không spawn baseline agent (bỏ agent tuần tự nặng nhất khỏi critical path); `nonDiscriminating` = carriedAnalyst; per-block `baseline: n-a` (round này không đo — trung thực), section Analyst ghi "carried từ round N". Round nào cũng ghi dòng baseline (chạy thật hoặc carried) để chuỗi hash liền mạch.
- Default `runBaseline !== false` → chạy (tương thích ngược args cũ).

## P3 — Memoize judge panel

- Main loop mỗi round tính `inputsHash` per judgment eval = sha256(question + nội dung từng file `inputs`). Đọc dòng `kind:"panel"` cuối của eval trong run-log: hash trùng → `carriedPanels[]` entry {evalId, proposal, votes, from_round}; khác/thiếu → judge fresh. Item UNCERTAIN-chờ-human với inputs không đổi → không chấm lại (câu trả lời ở Gate 2, không ở máy).
- Script: chỉ spawn 3 judge cho item fresh; `panels` = fresh + carried (đánh dấu `carried`); verdict routing GIỮ NGUYÊN trên tổng panels (T3/PENDING-JUDGMENT y cũ); report ghi "panel giữ từ round N — inputs không đổi", votes lens+verdict, rationale tham chiếu round gốc.
- Áp dụng mọi round ≥2 (kể cả fix round — hash là bằng chứng input không đổi).
